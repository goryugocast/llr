# Logic Spec: Toggle Intelligence (忖度ロジック)
**Authored by goryugo**

Markdownの柔軟性を活かしつつ、データの整合性を保つためのインテリジェントなトグル挙動の定義。

## 1. 判定マトリックス (Status × Time Count)

トグル実行時の行の「既存記号」と「時刻（HH:mm）の数」の組み合わせで挙動を決定する。

| 記号 (Status) | 時刻数 | 判定 | 期待されるアクション |
| :--- | :--- | :--- | :--- |
| `[ ]` / 無 | 0 | **新規開始** | 現在時刻で打刻開始 `- [/] NOW - ` |
| `[ ]` / 無 | 1 | **開始 or 復元** | ハイフンがあれば復元、なければ現在時刻で開始（既存時刻はメモ） |
| `[ ]` / 無 | 1(+Dur) | **実績補完** | `13:00 50m` ➔ 13:50終了の実績として `[x]` |
| `[ ]` / 無 | 2+ | **実績修復** | 時刻をソートし、実績ログ `[x]` として一撃完了 |
| `[/]` | 1 | **完了** | 現在時刻で終了打刻 |
| `[/]` | 2+ | **予定完了** | 行内の2つ目の時刻を終了時刻として採用し完了 |
| `[x]` | - | **リピート** | タスク名を複製し、現在時刻で新規開始 |

## 2. 数値パースの優先順位

1. **HH:mm (時刻パターン)**:
    - 24時間表記。行内のどこにあっても抽出。
    - 複数ある場合は、ソートして開始・終了に割り当てる。
2. **m (Minute) 単位**:
    - `30m` などの表記があれば、開始時刻に加算して終了時刻を算出。
3. **h (Hour) 表記 (隠し機能)**:
    - `1.5h` などを分に変換 (`1.5 * 60 = 90`)。
    - 小数は切り捨て（Integer truncate）。
4. **孤立した数字 (Smart Estimate)**:
    - 行末の ` 30` などを `(30m)` の見積もりとして扱う特殊パース。

## 3. 正規化（Normalization）

全てのトグルアクションは、実行後に行のフォーマットを以下に強制的に揃える。
- `- [ ] / [/] / [x]` のいずれかの記号で始まる。
- 時刻の前後には ` - `（ハイフンとスペース）を配置する。
- ユーザーが無意識に崩した記述を、ツールが「直す」ことでパースの安定性を高める。
- **操作対象の厳格化**: UIイベント（タップ等）を起点とするトグルは、必ずターゲットが所属する物理行を対象とする。判定に曖昧さが残る場合は実行を中止し、カーソル行へのフォールバック等による予測不能な副作用を回避する。

### 旧仕様メモ（廃止）
- `- [-]`（中断ステータス）は現行仕様では生成しない。
