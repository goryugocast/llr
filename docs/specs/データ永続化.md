# Data Persistence Specification

## Status
- **Task execution history persistence: Not Implemented / Out of Scope**
- **Debug trace JSONL: Implemented (Debug Mode only)**
- LLR は通常利用向けの JSONL / stats 永続ログ機能は提供しない。
- ただし、デバッグモード時のトレースログ（`llrlog/logs/debug/*.jsonl`）は例外として出力する。

## 1. Current Behavior
- タスクの実績は、ユーザーが編集する Markdown 行（主に `- [x] ...`）のみを正とする。
- `Toggle Task` / `Start Task` / `Stop Task` / `Interrupt Task` 実行時に、追加のログファイルは生成しない。
- 集計表示（ステータスバー）は、開いているノート内容からその場で再計算する。

## 2. Non-Supported Data Files
以下のファイルは**現行仕様では生成しない**。
- `llrlog/logs/YYYY.jsonl`
- `llrlog/stats.json`

## 2.1 Debug Trace Files (Exception)
- `llrlog/logs/debug/YYYY-MM-DD.jsonl`
- 用途: 実績保存ではなく、開発/検証用のイベントトレース
- 詳細: [[デバッグモード]]

## 3. Rationale
- 実測ログの保存方式は検討の結果、現行スコープから除外した。
- 機能の真実ソースを Markdown のみに絞り、挙動の一貫性と保守性を優先する。
