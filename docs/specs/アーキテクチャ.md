# System Architecture (システム構成)

## 1. Vault Structure (ディレクトリ構造)
```text
(Vault Root)
├── Daily work note        <--Current Workspace (legacy example: today.md)
├── routine/                    <--Routine Definitions (Master)
└── llrlog/             <--System Data (Current: routine/frontmatter only)
```

## 2. Core Components

### A. Day Boundary logic (日付境界・繰り越し)
**Status: Not Implemented**
- 指定された時刻（例: AM 4:00）を境界として、新しい一日を開始する。
- 実行時に現在のデイリーノート/作業ノートをアーカイブし、未完了タスクと今日のルーチンを新しく書き込む。

### B. Async Writer (非同期書き込み)
**Status: Out of Scope (Current)**
- JSONL等の外部ログ保存は現行仕様では実装しない。
- 実績の正は Markdown のタスク行とする。

### C. Discovery Logic (発展的紐付け)
- 構造の変化に追従するため、以下の3段階で判定を行う：
    1. **Explicit (@)**: 行内に `@[[routine/Name]]` があれば強制紐付け。
    2. **Implicit (Capture)**: タグや親フォルダから自動判定。
    3. **Learning**: ユーザーの選択を学習。

### D. Rebuild (再構築)
**Status: Not Implemented**
