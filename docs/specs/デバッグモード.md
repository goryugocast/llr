# Debug Mode Specification

## Status
- Implemented (v0.x, debug trace only)

## 1. Overview
- 目的: コマンド実行や内部の遅延処理（debounce）の時刻差を可視化する。
- デバッグモード有効時のみ、`Notice` と JSONL ログを出力する。
- 本仕様は「実績データの永続ログ」ではなく、開発/検証用のトレースログである。

## 2. Settings
- 設定画面に `Debug mode` トグルを追加する。
- 保存先: プラグイン設定ファイル（`loadData/saveData`）。
- デフォルト: `false`（無効）。

## 3. Notice Behavior (Obsidian標準アラート)
- デバッグモード有効時、対象イベントごとに `Notice` を表示する。
- `Notice` は種類に関係なくグローバルで抑制し、5秒間に1件まで表示する。
- 表示内容: ローカル時刻（`HH:mm:ss.SSS`）+ source + message + 短いデータ要約。
- 例:
  - `"[Debug 09:12:33.105] plugin command:start {...}"`
  - `"[Debug 09:12:43.210] routine-engine scheduleUpdate:timer-fired {...}"`

## 4. JSONL Debug Log

### 4.1 Path
- `llrlog/logs/debug/YYYY-MM-DD.jsonl`
- 補足: `routine/` 配下に置くと Base（リピートタスク一覧）の監視対象と干渉しやすいため、`routine/` 外に保存する。

### 4.2 Format
- 1イベント = 1行の JSON（JSONL）
- 追記専用（append）

### 4.3 Record Schema
- `timestamp`: ISO8601 (`toISOString()`)
- `localTime`: `HH:mm:ss.SSS`
- `source`: `"plugin"` | `"routine-engine"`
- `message`: イベント名
- `data`: 任意の補助情報（オブジェクト）

### 4.4 Example
```json
{"timestamp":"2026-02-26T00:12:33.105Z","localTime":"09:12:33.105","source":"plugin","message":"command:start","data":{"commandId":"toggle-task","commandName":"Toggle Task","startedAt":"2026-02-26T00:12:33.105Z"}}
```

## 5. Tracked Events (Current)
- コマンド開始/終了/エラー（LLR が `addCommand` で登録するコマンド）
- 既存 `debugLog(...)` で出している内部イベント
- RoutineEngine の遅延処理イベント:
  - `scheduleUpdate:scheduled`
  - `scheduleUpdate:cancel-existing`
  - `scheduleUpdate:timer-fired`
  - `scheduleUpdate:not-scheduled`
  - `processCompletion:start/done/fallback`
  - `updateNextDue:start/done`
  - `flushAll:start/done`

## 6. Intended Use (タイムラグ検証)
- 「タスク完了の時刻」と「routine note の `next_due` 更新の実行時刻」を比較できる。
- debounce により遅延しているか、キャンセルされているかを `Notice` と JSONL の双方で追跡できる。

## 7. Current Debug-Phase Runtime Setting
- RoutineEngine の更新遅延は、デバッグ段階では **0ms（実質即時）** に設定している。
- 理由: 過剰反応のデバッグを優先し、体感ラグを排除して挙動を観察するため。
- 将来、誤操作吸収（取り消し猶予）を再導入する場合は、遅延を再度増やす可能性がある。

## 8. Operational Notes (iCloud / Multi-device)
- iCloud Vault で複数デバイス（desktop / mobile）が同時にプラグインを有効化している場合、同じ debug JSONL に追記される。
- デバッグ中は、必要に応じて片側デバイスのプラグインをOFFにして、書き込み元を1つに絞る。
