## 0. この文書の前提

- 時間記録系のアプリを使った経験があり、考え方はすでに理解している人向け
- ここでは「このプラグインでどう操作するか」だけに絞る
- 用語は最小限にし、Markdown 行の実際の形で説明する

## 1. まず覚える操作（ほぼこれだけ）

### A. `Toggle Task`
現在行を、文脈に応じて開始/完了/再実行に進める基本操作。

### B. `Adjust Time (1m)`（補助）
現在行の時刻を「1分戻す」補正コマンド。微修正専用。

※ 実用性は高いが、まずは `Toggle Task` とセットで使う補助操作として覚えるとよい。

- 時刻が1つ（実行中など）: 開始時刻を `-1分`
- 時刻が2つ以上（完了など）: 終了時刻を `-1分`
- 完了行の `(Xm)` / `(見積 -> 実績)` は自動で再計算

例:

- `- [/] 09:00 - メール` -> `- [/] 08:59 - メール`
- `- [x] 09:00 - 09:30 (30m) メール` -> `- [x] 09:00 - 09:29 (29m) メール`

## 2. `Toggle Task` の基本状態遷移（最初はここだけ）

### A. 未着手を開始する

Before

`- [ ] タスク`

After

`- [/] HH:mm - タスク`

### B. 実行中を完了する

Before

`- [/] HH:mm - タスク`

After

`- [x] 開始 - 終了 (Xm) タスク`

### C. 完了済みを再実行する

Before

`- [x] 09:00 - 09:30 (30m) タスク`

After

- 元の完了行はそのまま
- 下に再実行用の行を追加して開始

追加される行（例）

`- [/] HH:mm - タスク`

## 3. `Adjust Time (1m)` の実用ルール

### A. 実行中（時刻が1つ）

- 開始時刻を 1分戻す

例:

`- [/] 09:00 - タスク` -> `- [/] 08:59 - タスク`

### B. 完了（時刻が2つ以上）

- 終了時刻を 1分戻す
- 実績時間も再計算する

例:

`- [x] 09:00 - 09:30 (30m) タスク` -> `- [x] 09:00 - 09:29 (29m) タスク`

## 4. モバイル操作（チェックボックス）

### A. 短押し

- `- [ ]` -> 開始
- `- [/]` -> 完了
- `- [x]` -> 何もしない

※ 設定 `Editor Checkbox Override` を OFF にすると、このクリック上書きは無効になり、Obsidian 標準のチェック動作に戻る。`Toggle Task` などのコマンドとショートカットはそのまま使える。

### B. 長押し（モバイル目安 450ms / デスクトップ目安 900ms）

- `- [ ]` -> 前の完了時刻で開始（見つからなければ現在時刻）
- `- [/]` -> 未着手へ戻す（未着手として解釈できる形に正規化。開始時刻1つ＋見積りを保持）
- `- [x]` -> 未着手へ戻す（未着手として解釈できる形に正規化。実績時刻は落とし、見積りを保持）

## 5. 実用ワークフロー（最短）

### A. その場で記録する（リアルタイム）
1. 行を作る（`- [ ] タスク` でも平文でもよい）
2. `Toggle Task` で開始
3. 終わったら `Toggle Task` で完了
4. 1分ズレたら `Adjust Time (1m)` を必要回数だけ押す

### B. 後からまとめて記録する（後追い）
1. `13:00 読書 50m` のように時刻＋所要時間で書く
2. `Toggle Task` で整形して完了ログ化
3. 端数や押し遅れは `Adjust Time (1m)` で補正

### C. 連続タスクの開始時刻を揃える
1. 未着手行にカーソル
2. `Start Task (Align to Previous Completion)` を使う
3. その日の完了行全体で最も直近の終了時刻で開始（見つからなければ現在時刻）

## 6. `Toggle Task` の整形機能（便利機能）

`Toggle Task` は状態遷移だけでなく、入力整形にも使える。

### A. 見積もり整形

Before

`タスク 30`

After

`- [ ] タスク (30m)`

### B. 後追い記録の整形（時刻 + 所要時間）

Before

`13:00 読書 50m`

After

`- [x] 13:00 - 13:50 (50m) 読書`

## 7. よく使う補助コマンド（用途ベース）

### A. 開始/完了の強制
- `Start Task (Force)`: 状態に関係なく開始に寄せたい時
- `Stop Task (Force)`: 実行中を強制完了したい時

### B. やり直し・複製
- `Reset Task (Keep Estimate)`: 未着手として解釈できる形へ戻す（見積り優先で保持）
- `Duplicate Task`: 行を未着手で複製する（自動開始しない）
- `Interrupt Task`: 今の区間を完了として確定し、続きの未着手行を作る
  - 見た目は通常の完了行（特殊マークなし）
- `Retro Complete Task`: 後追い記録を完了形に確定したい時の明示コマンド
- `Fix Duration Drift (All Completed Tasks)`: 完了行 duration のズレを一括補正

### C. ルーチン挿入
- `Insert Routine`: `routine/` の繰り返しノートから、当日分を現在ノート末尾へ挿入
- `Skip Task (Log Only)`: `- [ ]` と `- skip:` を相互変換して当日表示の ON/OFF を切り替える

## 8. 行の見た目（最低限）

- 未着手: `- [ ] タスク (30m)`
- 実行中: `- [/] HH:mm - タスク`
- 完了: `- [x] 開始 - 終了 (実績m) タスク`
- 完了（見積差分あり）: `- [x] 開始 - 終了 (見積m -> 実績m) タスク`

## 9. モバイルでのおすすめ運用

- ボタンはまず2つで十分:
  - `Toggle Task`
  - `Adjust Time (1m)`
- 微修正は `Adjust Time (1m)` 連打で処理
- 大きい補正だけ `Start Task (Align to Previous Completion)` を使う

## 10. つまずきやすい点

- `Toggle Task` は単純なチェック切替ではない（開始/完了/整形の複合動作）
- `Duplicate Task` は未着手で複製される（自動開始しない）
- `Interrupt Task` は「特殊状態」を残さず、完了行 + 続きの未着手行を作る
