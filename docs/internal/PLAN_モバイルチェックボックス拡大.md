# PLAN: モバイルチェックボックス拡大

## 目的

モバイル環境で、デイリーノート上のタスク行が押しづらい問題を緩和する。

- 設定で ON/OFF できるようにする
- モバイル時のみ有効にする
- Apple 標準のタップしやすさに寄せる
- 既存のタスク状態遷移ロジックはなるべく変更しない

## 現状の課題

- モバイルの checkbox が小さく、リンク文字列と近い
- 見た目の小ささと実際の当たり判定の狭さが一致しており、タップしづらい
- タスクの合間に文章や見出しが入る構成では、視覚的にも操作領域が詰まって見える

## 実装方針

段階的に進める。

### Phase 1: CSS のみ

まずは見た目の改善だけを行う。

- 新しい設定を追加する
  - 例: `mobileLargeCheckboxEnabled`
- `Platform.isMobile` かつ設定 ON のときだけ、専用クラスを有効化する
- CSS で以下を拡大する
  - checkbox の見た目サイズ
  - task 行の文字サイズ
  - 行高
  - checkbox 周辺の余白
- 対象は task 行のみ
  - 通常の本文や見出しは変更しない

この段階では、クリック/タップ判定ロジックは変更しない。

### Phase 2: 最小限の判定調整（必要時のみ）

Phase 1 の体感でまだ押しづらい場合のみ、ヒット判定を少し緩める。

- 対象は `getCheckboxAtPoint()` 周辺に限定する
- `Platform.isMobile` かつ設定 ON のときだけ適用する
- 変更候補
  - `isCoordInsideElement()` の padding を拡大
  - モバイル拡大モード専用の padding 値を導入

ここでも、状態遷移や routine 更新などの中核ロジックは変更しない。

## 影響範囲

比較的小さい。

- 追加:
  - 設定項目
  - モバイル拡大モード用の CSS
- 条件付き変更:
  - モバイル時の UI クラス付与
- 必要時のみ:
  - checkbox のヒット判定

以下は原則として触らない。

- `- [ ] -> - [/] -> - [x]` の遷移仕様
- タスク変換ロジック
- ルーチン処理
- 時刻計算

## UI の意図

- 「押せそうに見える場所」と「押ける場所」を近づける
- ただし、編集領域は守る
- 文字まで広くトグル領域にしない
- 上下方向より、横方向と行高の改善を優先する

## 確認方法

- モバイルで設定 OFF:
  - 既存表示のまま
- モバイルで設定 ON:
  - task 行のみ大きく見える
- デスクトップ:
  - 設定に関係なく既存表示のまま
- Phase 2 実施時:
  - 誤爆が増えていないこと
  - 本文編集がしづらくなっていないこと
