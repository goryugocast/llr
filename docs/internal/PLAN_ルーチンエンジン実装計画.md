# Routine Engine (リピートタスク) 実装計画書
**STATUS: COMPLETED (2026-02-22)**

## 1. 目的
ノートの YAML Frontmatter に設定されたリピート規則を読み取り、タスク完了時に次回の予定日 (`next_due`) を自動計算して上書き更新する。特に**「正本を確実に見つけ、正しく更新する」**基本動作の信頼性を担保する。

## 2. 実装ポリシー
- **Obsidian標準APIの徹底活用**: `metadataCache` や `processFrontMatter` を優先し、自作のパース処理を最小限に抑える。
- **複数対応**: 1行に複数のルーチンリンクがある場合でも、漏れなくすべて処理する。
- **階層制限**: シンプルな運用のための `tc/` フォルダ直下（サブフォルダ不可）という制限を設ける。

## 3. 具体的な変更内容 (Phase 1)

### 3.1. リンク抽出の拡張
- `routine-engine.ts` の `extractLinkText` を、行内の**すべての WikiLink を配列で返す**ように変更する。
- 各リンクに対して `resolveRoutineFile` を実行し、対象ファイルをリストアップする。

### 3.2. フォルダ・階層チェックの強化
- `resolveRoutineFile` において、`ROUTINE_FOLDER` (`tc/`) の**直下**にファイルがあるかを確認するロジックを厳密化する（`/` の数などで判定）。

### 3.3. 日付計算のフォールバック
- `processCompletion` において、`calculateNextDue` がエラーを投げる、あるいは規則が取得できなかった場合の処理を追加。
- フォールバックアクションとして、基準日の**翌日（+1日）**を `next_due` にセットする。
