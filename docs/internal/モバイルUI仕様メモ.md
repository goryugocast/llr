# モバイルUIインタラクション：現状仕様 (2026-02-21)

**Status: Historical Snapshot**  
[[UI操作仕様]] の補助資料。モバイル不具合対応時点の実装メモとして保持する。

本ドキュメントは、モバイル環境におけるタスク・トグルの安定性を確保するために実装された「座標ベース判定」および「多重行解決戦略」の現状を記録するものです。

## 1. タップ判定（ヒットテスト）
モバイル環境において、Live Preview のチェックボックス（ウィジェット）を確実に操作し、かつテキスト編集（カーソル移動）を妨げないために以下の判定ロジックを採用しています。

- **判定方式**: 画面上の物理座標 (`x, y`) によるエリア照合。
- **ヒットエリア**: 
    - 取得したチェックボックス要素の `getBoundingClientRect()` に対して遊びを追加。
    - **遊び（Padding）**: モバイル = 20px, デスクトップ = 5px。
- **イベントフロー**:
    1. `pointerdown` / `click` イベントを捕捉。
    2. 座標がヒットエリア外であれば、即座に `return` し、Obsidian 標準の挙動（カーソル移動）を許可する。
    3. ヒットエリア内であれば、`preventDefault()` で標準挙動を中断し、Project-TC のトグルロジックを実行する。

## 2. 行解決戦略 (resolveLineIndex)
タップされた座標がエディタ上のどの行（行番号）に該当するかを特定するため、以下の3段階のフォールバックを実装しています。

1. **Strategy 1: DOM属性直接取得**
   - ターゲット要素の親を遡り、`.cm-line` 等にある `data-line` 属性を読み取る。
2. **Strategy 2: CodeMirror 6 API (推奨パス)**
   - `editor.cm.posAtCoords({ x, y })` を使用。
   - 座標をドキュメントのオフセットに変換し、Obsidian の `offsetToPos` で行番号を取得。
3. **Strategy 3: 最終手段 (Visual Scan)**
   - 上記が失敗した場合、エディタ・コンテナ内の全 `.cm-line` 要素を走査。
   - タップされた座標 `y` が要素の `top` 〜 `bottom` の範囲内に収まっている要素を特定し、その `data-line` を採用する。

## 3. レンダリングの安定化
- **非同期更新 (`setTimeout`)**:
  タップイベントの処理（テキスト書き換え）を `setTimeout(..., 0)` で次ティックに逃がすことで、Obsidian 本体のクリック処理との競合を防ぎ、Live Preview のウィジェットの状態（`data-task`）が正しく更新されるようにしている。
- **置換方式**: `editor.replaceRange` を使用し、行全体の置換を明示することで、DOMの再構築を確実にする。

## 4. CSSカスタマイズ
- `/` (進行中) と `-` (中断) のカスタムアイコンを表示するために背景をリセット。
- `x` (完了) になった瞬間に標準のチェックマークが表示されるよう、スタイルの強制復元を行っている（Live Preview のウィジェット更新遅延対策）。
